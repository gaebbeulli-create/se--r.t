# RiderFlex 배포 가이드 (GitHub Pages + Supabase)

## 구조
- **GitHub Pages**: index.html, vehicle.html, admin.html (무료 정적 호스팅)
- **Supabase**: DB + 이미지 Storage (무료 티어로 충분)
- **server.js 없음** — 브라우저에서 Supabase 직접 연결

---

## STEP 1: Supabase 설정

### 1-1. 프로젝트 생성
1. [supabase.com](https://supabase.com) → 회원가입 (GitHub 계정 가능)
2. **New Project** → 지역: **Northeast Asia (Seoul)**

### 1-2. SQL 실행 (테이블 + 스토리지 생성)
1. 좌측 메뉴 **SQL Editor → New query**
2. `supabase-setup.sql` 내용 붙여넣기 → **Run**

### 1-3. API 키 3개 메모
**Settings → API** 에서:
- **Project URL** (예: `https://abcdefgh.supabase.co`)
- **anon public** key (긴 문자열 — index.html, vehicle.html에 사용)
- **service_role** key (더 긴 문자열 — admin.html에만 사용)

> ⚠️ service_role key는 admin.html에만 넣으세요.
>    admin.html URL은 외부에 공유하지 마세요.

---

## STEP 2: HTML 파일에 키 입력

3개 파일 상단의 `<script>` 블록에서 아래 값을 교체:

**index.html, vehicle.html** (2개 파일):
```javascript
const SUPABASE_URL = 'https://abcdefgh.supabase.co';   // ← 교체
const SUPABASE_ANON_KEY = 'eyJhbGci...';               // ← 교체
```

**admin.html** (1개 파일):
```javascript
const SUPABASE_URL = 'https://abcdefgh.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGci...';
const SUPABASE_SERVICE_KEY = 'eyJhbGci...';   // ← service_role key
const ADMIN_PASSWORD = '원하는비밀번호';        // ← 교체
```

---

## STEP 3: GitHub Pages 배포

### 3-1. GitHub 저장소 생성
1. [github.com](https://github.com) 로그인
2. 우측 상단 **+** → **New repository**
3. Repository name: `riderflex` (또는 원하는 이름)
4. **Public** 선택 → **Create repository**

### 3-2. 파일 업로드
저장소 페이지에서 **Add file → Upload files** 클릭:
- `index.html`
- `vehicle.html`
- `admin.html`

**Commit changes** 클릭

### 3-3. GitHub Pages 활성화
1. 저장소 상단 **Settings** 탭
2. 좌측 **Pages** 클릭
3. Source: **Deploy from a branch**
4. Branch: **main** / **(root)**
5. **Save**

### 3-4. 접속 주소 확인
약 1~2분 후:
```
https://[GitHub계정명].github.io/riderflex/
```

---

## STEP 4: 초기 설정

1. `https://[주소]/admin.html` 접속
2. `ADMIN_PASSWORD`에 설정한 비밀번호로 로그인
3. **사이트 설정** → 전화번호, 카카오톡 URL 입력 → 저장
4. **차량 관리** → 이미지 업로드 및 가격 입력

## 문제 해결

**이미지가 안 보여요**
→ Supabase Storage의 `vehicle-images` 버킷이 Public인지 확인
→ SQL의 storage 정책이 제대로 실행됐는지 확인

**상담 접수가 안 돼요**
→ Supabase의 `consult_requests` 테이블 RLS 정책 확인
→ anon 키로 INSERT가 허용되어 있는지 확인

**관리자에서 데이터가 안 불러와져요**
→ admin.html의 `SUPABASE_SERVICE_KEY`가 올바른지 확인
→ service_role key는 Settings > API > service_role 항목
